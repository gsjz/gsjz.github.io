# 操作系统概述

现代操作系统中最基本的两个特征是：Concurrence 并发和 Sharing 共享。另外两个是 Virtual 虚拟和 Asynchronism 异步。

用户可以通过 命令接口 和 系统调用 来使用计算机。

系统调用 System Call 是由操作系统提供给用户的，它只能通过用户程序间接使用。

系统调用的目的是请求系统服务。

命令接口可进一步分为 联机用户接口 和 脱机用户接口。

???- note "库函数 Library Function 与系统调用的区别和联系"

    | 维度       | 库函数          | 系统调用      |
    | -------- | ------------ | --------- |
    | 提供者      | 函数库（glibc 等） | 操作系统内核    |
    | 运行态      | 用户态          | 用户态 → 内核态 |
    | 是否切换特权   | 否（通常）        | 是（必须）     |
    | 接口稳定性    | 相对稳定         | 与内核版本强相关  |
    | 可移植性     | 高            | 低         |
    | 性能开销     | 较小           | 较大（上下文切换） |
    | 能否直接操作硬件 | 否            | 是（通过内核）   |

    库函数往往是系统调用的封装；并非所有库函数都会调用系统调用；一个库函数可能封装多个系统调用
