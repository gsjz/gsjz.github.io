    commit 规范¶

    之前 commit 的时候比较摆烂，最近学习一下规范

    feat：新功能

    fix：修 bug

    docs：只改文档

    style：格式、空格、换行等，不改逻辑

    refactor：重构（不加新功能、不修 bug）

    test：测试相关

    chore：构建脚本、依赖升级等杂项
